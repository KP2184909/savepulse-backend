# SavePulse Product Brief

## Core idea

SavePulse helps people avoid exchanging money blindly.

The product does not tell users how to trade. It helps them understand whether the current exchange rate looks favorable or unfavorable compared with recent historical data.

## Target users

Primary users:
- Travelers
- People holding foreign currency
- Freelancers receiving USD or foreign currency
- International students
- SME import/export operators
- Gold or currency watchers

## Main emotional pain

Users are afraid of exchanging at a bad time and regretting it later.

The product should reduce regret, not promise profit.

## Main screen copy

Use these decision states:

### Good state

Headline:
Good time to exchange

Subtext:
Today’s rate looks relatively favorable compared with recent historical data. If you need to exchange soon, this may be a reasonable time.

CTA:
Set an alert in case the rate improves further

### Wait state

Headline:
Wait for now

Subtext:
Exchanging now may be less favorable. Today’s rate is still relatively expensive compared with recent historical data. If you are not in a hurry, consider waiting.

CTA:
Alert me when the rate improves

### Neutral state

Headline:
Not urgent yet

Subtext:
The rate is not clearly favorable or unfavorable. If you need to exchange soon, consider exchanging partially. If you can wait, set an alert.

CTA:
Set a free alert

## FOMO examples

Use mild FOMO based on historical examples.

Good:
- If you exchanged 3 months ago, the difference could have been about 3.1%.
- A small exchange-rate move can quietly change your real cost.
- Many people compare exchange shops, but forget to compare timing.
- Good rate windows do not stay open forever.

Avoid:
- You missed guaranteed profit.
- Use SavePulse and make money.
- Guaranteed better exchange rate.
- Never lose from exchange rates.

## Interface rules

- The first thing users see should be the decision, not a chart.
- Use charts only as supporting evidence.
- Use flags and currency names before technical pair symbols.
- Show technical pair names only as “Advanced pair: EUR/USD”.
- Avoid complex words like volatility, percentile, regime, signal, backtest in beginner UI.
- Keep the main CTA focused on free alerts.

## Subscription logic

Free users should see:
- One watchlist
- Basic decision state
- Delayed or limited alerts
- Generic historical example

Paid users unlock:
- Real-time alerts
- Multiple watchlists
- Personalized amount-based examples
- Timing window
- Confidence explanation
- More assets such as gold and BTC
